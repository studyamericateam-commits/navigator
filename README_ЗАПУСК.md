# Запуск Навигатора на studyglobal.tech — пошаговая инструкция

**Для владельца StudyGlobal · 10.07.2026 · Время на всё: ~40 минут, навыки программирования не нужны**

## Что получится в итоге

Работающая платформа на https://studyglobal.tech с видео-онбордингом, всеми 4 чекпоинтами, документами и админ-кабинетом. Одно известное ограничение первого этапа: живой ИИ (портрет Веги, чат, ИИ-версия Strategy Book) заработает на этапе 2 (Supabase-шлюз) — до этого продукт полностью проходим, а ИИ-блоки используют встроенные запасные тексты. Для демо команде и вёрстки-проверки этого достаточно.

## Шаг 1. Регистрации (10 мин)

1. **GitHub.com** — «Sign up», обычная почта. Это хранилище кода.
2. **Vercel.com** — «Sign up» → «Continue with GitHub» (важно: входить через GitHub). Это хостинг.

## Шаг 2. Код на GitHub (10 мин, без git)

1. GitHub → «+» справа сверху → **New repository** → имя `navigator` → Private → Create.
2. На странице пустого репозитория: ссылка **«uploading an existing file»**.
3. Распакуйте архив `navigator-app.zip` (прислан вместе с этой инструкцией) и перетащите в окно загрузки **всё содержимое папки** (файлы и папки src, public, package.json и т.д. — именно содержимое, не саму папку navigator-app).
4. Кнопка **Commit changes**.

## Шаг 3. Деплой на Vercel (5 мин)

1. Vercel → **Add New… → Project** → напротив репозитория `navigator` кнопка **Import**.
2. Ничего не меняя (Vercel сам поймёт Vite), жмите **Deploy**.
3. Через ~1 минуту получите адрес вида `navigator-xxxx.vercel.app` — платформа уже живая, проверьте её.

## Шаг 4. Домен studyglobal.tech (10 мин + ожидание DNS)

1. В Vercel: проект → **Settings → Domains** → введите `studyglobal.tech` → Add. Vercel покажет требуемые DNS-записи.
2. В личном кабинете **reg.ru**: Мои домены → studyglobal.tech → **DNS-серверы и управление зоной** (если предложит — «использовать DNS reg.ru»). Добавьте записи:
   - Тип **A**, имя `@`, значение `76.76.21.21`
   - Тип **CNAME**, имя `www`, значение `cname.vercel-dns.com.`
3. Подождите от 10 минут до пары часов (обновление DNS). Vercel сам выпустит SSL-сертификат — сайт откроется по https.

## Шаг 5. Проверка доступности из России (5 мин)

1. Откройте https://check-host.net → вкладка HTTP → введите `https://studyglobal.tech` → убедитесь, что российские точки зелёные.
2. Попросите 2–3 человек на разных операторах (МТС/Билайн/домашний интернет) открыть сайт с телефона.
3. Если где-то не открывается — сообщите Claude: включим план Б (перенос статики на Timeweb Cloud, ~1 час, домен просто перенацелим).

## Как обновлять платформу дальше

Claude присылает новый `src/App.jsx` → GitHub → откройте файл `src/App.jsx` в репозитории → иконка карандаша (или «Upload files» с заменой) → Commit. Vercel задеплоит сам за минуту. Всё.

## Этап 2 (следующий): данные и живой ИИ

Supabase (Франкфурт): авторизация magic-ссылками по email оплаты (связка с GetCourse — см. «ТЗ_интеграция_GetCourse_доступ.md»), база ответов учеников (админ-кабинет начнёт видеть реальных людей), и ИИ-шлюз — ключ Anthropic живёт на сервере, Вега оживает. Регистрация на supabase.com — тоже через GitHub; дальше пошагово с Claude.

## Частые вопросы

- **Сколько стоит?** Vercel и GitHub — бесплатно на нашем масштабе. Supabase — бесплатно до ~50 тыс. пользователей БД. Платить будете только за токены ИИ (~5–7 ₽/ученик) и рассылку писем.
- **Vercel просит карту?** На бесплатном тарифе — нет. Если попросит верификацию — подойдёт международная карта компании.
- **Нужен ли хостинг reg.ru?** Нет, reg.ru остаётся только регистратором домена (продление раз в год).
- **Что с «noindex»?** В коде стоит запрет индексации поисковиками — пока платформа не для случайных гостей. Уберём при публичном запуске.
